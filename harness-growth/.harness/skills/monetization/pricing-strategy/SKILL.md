# Pricing Strategy — 定价策略

## 铁律
- 定价必须基于**价值**，不是成本加成
- 必须同时考虑**竞品定价**和**用户支付意愿**
- 必须设计**价格档位**（免费/基础/专业/企业），不是单一价格

## 流程

1. **三视角定价分析**

   ### 价值定价
   - 产品为用户创造多少价值？（节省时间/增加收入/降低成本）
   - 用户愿意为价值支付多少？（价值的 10-30%）
   ```
   价值量化:
   - 节省时间: X 小时/月 × ¥Y/小时 = ¥Z/月
   - 增加收入: +X% 转化 × ¥Y 收入 = ¥Z/月
   定价参考: 价值的 10-30% = ¥A-¥B/月
   ```

   ### 成本定价
   - 边际成本 + 固定成本分摊
   - 毛利率目标（SaaS 通常 > 70%）
   ```
   成本: ¥X/用户/月
   目标毛利: 80%
   最低定价: ¥X / (1-0.8) = ¥Y
   ```

   ### 竞品定价
   ```
   | 竞品 | 定价 | 功能 | 差异 |
   |------|------|------|------|
   | A | ¥99/月 | 全功能 | 我们更优 |
   | B | ¥49/月 | 基础 | 我们更全 |
   | C | ¥199/月 | 高级 | 我们更便宜 |
   ```

2. **Van Westendorp 价格敏感度测试**（如有调研数据）
   - 太便宜（质疑质量）: ¥X
   - 便宜（划算）: ¥Y
   - 贵（需考虑）: ¥Z
   - 太贵（不会买）: ¥W
   - 最优价格区间: [便宜-贵] = [¥Y, ¥Z]

3. **价格档位设计**
   ```
   | 档位 | 价格 | 功能 | 目标用户 |
   |------|------|------|---------|
   | 免费 | ¥0 | 基础功能 | 试用/传播 |
   | 基础 | ¥49/月 | 核心功能 | 个人用户 |
   | 专业 | ¥99/月 | 全功能 | 专业用户 |
   | 企业 | ¥299/月 | 全功能+支持 | 团队/企业 |
   ```

4. **免费策略设计**
   - 功能门控：免费版限制哪些功能？
   - 用量门控：免费版限制多少用量？
   - 试用策略：限时试用全功能 vs 永久免费基础版
   - 信用卡门控：是否要求信用卡？（降低试用量但提高付费质量）

5. **产出定价方案**
   写入 `docs/operations/pricing-strategy.md`

## 禁止事项
- 不只看成本定价（忽略用户感知价值）
- 不做单一价格（缺少价格锚点和选择）
- 不忽略竞品定价（用户会对比）
- 不频繁调价（损害信任，建议年度调价）
