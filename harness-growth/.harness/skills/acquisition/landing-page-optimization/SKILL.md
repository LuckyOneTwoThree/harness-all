# Landing Page Optimization — 落地页优化

## 铁律
- 落地页必须与广告创意**信息一致**——广告承诺什么，落地页就给什么
- 每个落地页只聚焦**一个 CTA**——多个 CTA 会分散注意力
- 表单字段越少越好——每多一个字段转化率降 5-10%

## 流程

1. **定义落地页目标**
   - 主要转化目标：注册/试用/购买/下载/预约
   - 次要目标：了解更多/订阅/分享
   - 一页一主 CTA

2. **价值主张设计**
   ```
   标题（H1）: [核心价值，< 15 字]
   副标题: [具体说明，< 30 字]
   价值点1: [功能/好处]
   价值点2: [功能/好处]
   价值点3: [功能/好处]
   社会证明: [用户数/评分/案例/Logo]
   CTA: [行动号召]
   ```

3. **页面结构设计**
   ```
   Above the fold（首屏可见）:
   - H1 标题 + 副标题
   - 主 CTA
   - 信任徽章/社会证明

   Below the fold:
   - 功能详解（图文）
   - 用户案例/评价
   - FAQ
   - 再次 CTA
   ```

4. **表单优化**
   | 字段数 | 转化率影响 | 适用场景 |
   |--------|-----------|---------|
   | 1-3 字段 | 基线 | 免费试用/注册 |
   | 4-6 字段 | -10~20% | 付费试用/预约 |
   | 7+ 字段 | -30%+ | B2B 线索/高端服务 |

   - 只留必要字段
   - 使用智能默认值
   - 支持社交账号登录（减少输入）
   - 实时验证（减少提交失败）

5. **信任元素**
   - 用户 Logo / 数字
   - 评分 / 评价
   - 安全认证 / 隐私承诺
   - 退款保证 / 免费试用

6. **性能优化**
   - 加载速度 < 3s（每多 1s 转化率 -7%）
   - 移动端适配
   - 图片压缩 + Lazy load

7. **A/B 测试建议**
   ```
   | 测试项 | 变体A | 变体B | 假设 |
   |--------|-------|-------|------|
   | 标题 | 功能描述 | 价值描述 | 价值导向转化更高 |
   | CTA | "注册" | "免费开始" | 利益导向 CTR 更高 |
   | 表单 | 5字段 | 3字段 | 少字段转化更高 |
   ```

8. **产出落地页方案**
   写入 `docs/operations/landing-page-spec.md`

## 禁止事项
- 不做广告与落地页不一致的页面（跳出率高）
- 不放多个 CTA（分散注意力）
- 不设计过长表单（除非 B2B 高价值线索）
- 不忽略移动端（移动流量占比 50%+）
